Current runtime documentation · 17 July 2026

This page describes the pipeline currently used on main. The section editor is on by default. The older multi-step duplicate detector still exists in code, but it does not run on the default path. The removed enrichment system, arbiter, sentence statuses, heat maps, and restore queue do not run either.

Technical workflow

One video from raw input to reviewed output

Processing layers

Audio layer FFmpeg extracts audio. Spectral gating reduces noise. Volume analysis finds silence and loud disruptions.
Speech layer ElevenLabs Scribe v2 creates Croatian text and word timestamps. Pause rules split sentences. Gemini corrects spelling.
Decision layer GPT-5.6 Sol finds cuts. Rule-based checks catch clear local corrections and audio false starts. Gemini checks asides. The edit decision list (EDL) keeps each cut's source and moves boundaries to quiet audio.
Review layer A FastAPI + React app lets the reviewer edit any source-time range. It provides Transcript and Compare views, a review.v4 payload, and separate reviewed EDL and video files.

Sequential pipeline

Processing stages

Each stage feeds the next one. If a section-editor call fails, the pipeline retries it and can switch from OpenRouter to OpenAI. If both fail, that section is left unedited. The evaluation report records the failure.

1

Audio extraction and normalization

The pipeline finds the first audio stream and extracts it as a mono, 16-bit PCM WAV. It keeps the original sample rate. This WAV is used for analysis, while the original video remains the picture source for rendering.

ToolingFFmpeg and ffprobe.
OutputRaw mono WAV in the temporary workspace.
PurposeGive every audio analysis step the same stable input.
2

Noise reduction and silence map

Non-stationary spectral gating reduces steady background noise. The default setting is mild so it does not damage speech. FFmpeg then finds quiet regions. The pipeline treats the remaining audio as speech and adds padding around it.

AlgorithmNon-stationary spectral gating via noisereduce.
Silence threshold-40 dB, minimum 3 seconds by default.
Speech padding500 ms before and after detected speech regions.
3

Speech-to-text and correction

ElevenLabs turns the cleaned audio into Croatian text with a timestamp for every word. Cough and laughter markers are stored separately from spoken words. Punctuation and long pauses split the words into sentences. Gemini fixes obvious spelling mistakes without changing timestamps, grammar, speaking style, or filler words.

STT modelElevenLabs Scribe v2, language hr, word timestamps.
Audio eventsEnabled by default and stored separately.
Correction modelGemini 2.5 Flash, structured output, up to 5 passes.
Sentence splitPunctuation plus pause split at 1.5 seconds by default.
4

Text editing and supporting detectors

GPT-5.6 Sol reads large transcript sections and names the exact text it wants to remove. It labels each span as a retake, false start, stutter, filler, or redundant text. Code checks the text and maps it to timestamps. Separate rule-based detectors handle very clear local corrections and audio false starts. Gemini checks short possible asides.

Main editorGPT-5.6 Sol through OpenRouter; direct OpenAI fallback.
Section sizeTarget 1,200 words, maximum 2,000, two-sentence overlap.
Model contractSol names text; code maps it to existing word timestamps.
Local correctionsRapidFuzz rules; no extra model call.
Aside verifierGemini 2.5 Flash only for short candidates near silence or STT audio events.
Inactive pathThe older lexical and embedding duplicate detector is skipped while section_editor.enabled is true.
5

Build the EDL and adjust cut boundaries

The pipeline combines silence regions and accepted cut flags into one EDL (edit decision list). A flag can remove a full sentence or only selected words. If one gap contains both silence and a text cut, it is split so each part keeps the correct reason. An RMS (root-mean-square) volume envelope then moves each cut boundary to nearby quiet audio. It may not consume more than half of a neighboring word.

Decision fieldsstart, end, action, reason, confidence, note.
Cut sourceText cuts keep their reason and Croatian note; real silence stays labeled silence.
Audio envelope25 ms RMS frames, 10 ms hop, persisted as <stem>.audio.json.
No arbitrationThe EDL is consumed directly; there is no enrichment pass after it.
6

Automatic render

FFmpeg renders the automatic EDL as a draft MP4. It takes video from the raw recording and audio from the denoised WAV. It trims every kept segment, resets that segment's timestamps, and joins the segments. Short audio fades reduce clicks and harsh transitions at each join.

RendererFFmpeg filter graph with trim, atrim, concat, and audio fades.
Default codeclibx264, CRF 28, ultrafast preset.
Splice smoothing30 ms audio fade in and fade out at segment boundaries.
StatusProduction render settings still need final validation.
7

Human review and reviewed render

A local FastAPI + React app shows the video, waveform, transcript, and cut ranges. The waveform and transcript both edit the same list of source-time ranges. The reviewer can restore a cut, add a new cut anywhere, preview the result, save a separate reviewed EDL, and render a reviewed MP4. For QA fixtures, Compare shows pipeline cuts and human cuts together. The raw video and automatic EDL are never overwritten.

Schemareview.v4; cut_ranges is the source of truth for the current edit.
ViewsTranscript and Compare; no annotation queue or sentence-status legend.
Reviewed artifactSeparate reviewed EDL, not an overwrite of the automatic EDL.
Reviewed outputRendered from the review UI with the _reviewed suffix.

Decision orchestration

How keep and cut decisions are made

Sol decides which words appear removable. Rule-based code then checks the proposed spans, adds a few cuts supported by strong mechanical evidence, keeps the reason for every cut, and chooses safe audio boundaries. Every accepted proposal becomes an automatic cut in the EDL.

Sol section editor

Sol reads the surrounding passage, not just one isolated pair of sentences.

  • It returns exact copied text labeled retake, false_start, stutter, filler, or redundant.
  • It may edit only sentences owned by the section. Overlap sentences provide context but cannot be changed.
  • Invalid structured output is retried up to three times per section.
  • OpenRouter is primary; direct OpenAI is the default fallback for Sol.
  • If both providers fail, that section stays unedited and is counted as failed. The rest of the video continues.
target 1200 words max 2000 words overlap 2 sentences 3 attempts

Span validation and local corrections

Code turns Sol's text into timestamped cuts. A small rule-based detector then adds only clearly provable local correction chains, without another AI call.

  • The proposed text must match at least 80% of one continuous word span in the named sentence.
  • If it covers at least 90% of the sentence, the whole sentence is cut. Otherwise only the matched words are cut.
  • A full retake under four words is rejected. A retake with a named matching sentence over 60 seconds away is also rejected.
  • Every redundant proposal is rejected because the UI has no review-suggestion queue.
  • A rule-based local correction needs a visibly cut-off middle attempt and very similar endpoints no more than ten seconds apart.
span match ≥ 0.8 whole sentence ≥ 0.9 word-level trim local flags 0.99

Asides and acoustic disruptions

Noise and short comments outside the lesson are different from retakes, so separate detectors handle them.

  • A possible aside must be near a long silence or have an STT audio-event marker.
  • Gemini 2.5 Flash decides whether that short candidate belongs to the lesson.
  • RMS energy analysis looks for loud non-speech bursts in gaps between transcribed words.
  • The audio false-start rule cuts a short stranded phrase when a disruption and long pause are followed by new speech soon afterward.
RMS frame 25 ms hop 10 ms noise floor pct 10 threshold +22 dB

Decision policy

Sol chooses the text; code checks and applies it

Sol decides what the words mean and which text may be removed. Code does not make that language judgment again. It checks the proposal, blocks known unsafe cases, maps the text to time, and records the evidence for review and QA.

What Sol controls

Sol chooses the cut type, exact copied text, and Croatian explanation. For a retake, it also names the later sentence that should remain.

  • Accepted model flags use confidence 0.9.
  • After the span check, partial false starts, stutters, and filler become automatic cuts.
  • No second model reviews or annotates Sol's choices.
  • Therefore, two model runs can still make different automatic cuts.
structured output confidence 0.9

Guardrails that outrank the model

Fixed code checks every proposal and can reject it.

  • Text that cannot be matched and sentence indices outside the editable section are rejected.
  • Whole-sentence retakes shorter than four words are rejected.
  • A named matching sentence is rejected if it is earlier or more than 60 seconds away.
  • Risky redundant removals stay kept. They do not become hidden suggestions.
protect_min_words 4 retake_max_gap_s 60 reject redundant

Measured against human editors

The QA harness compares pipeline cuts with 98 videos edited by humans. We use those scores to decide whether a pipeline change is safe.

  • Human cuts are measured by word coverage in the human-edited video, not by matching whole sentences.
  • Word-level precision, recall, and F1 correctly score partial sentence trims.
  • eval-section-editor tests models on cached transcripts and saves EDLs, proposal traces, health data, run metadata, and reports.
  • The development diff view marks pipeline and human cuts on the same raw transcript.
  • LangSmith tracing can record every LLM call when enabled.
  • Failure patterns are catalogued in MISTAKE_PATTERNS.md and drive the roadmap.

Active architecture

Section editor: Sol names text, code turns it into cuts

This is the default text-editing path when section_editor.enabled = true. Normal process and batch runs skip the older sentence-pair detector.

Design

Sol reads large transcript sections and copies the exact text it wants to remove. Fixed code converts that text to timestamps.

  • The transcript is divided into sections of about 1,200 words at pause boundaries. Neighboring sentences overlap for context, but each sentence belongs to only one editable section.
  • Sol returns whole sentences or partial spans labeled retake, false start, stutter, filler, or redundant.
  • If the copied text cannot reach an 80% match inside the named sentence, the proposal is rejected.
  • Code maps accepted text to existing word timestamps. Sol never creates or changes timestamps.
  • Time-gap and short-retake guardrails block risky proposals. redundant proposals stay kept because there is no suggestion queue.
enabled by default span match ≥ 0.8 protect < 4 words retake gap ≤ 60 s

Review payload semantics

Review changes the saved cut decisions, not the raw video. Those decisions are source-time ranges linked to transcript words and timeline segments.

  • Each word stores its source time, original AI keep/cut state, current state, reason, confidence, and nearby quiet split points.
  • Each sentence stores its current and original action, kept-word coverage, reason, confidence, note, and words.
  • cut_ranges is the source of truth. It comes from the reviewed EDL when one exists; otherwise it comes from the automatic EDL.
  • Reviewer changes are saved as any source-time range, not only whole words. Saving by word index still exists only for older API clients.
  • The reviewed EDL is rebuilt by merging all cut ranges and keeping everything else in the source duration.
  • The reviewed render uses the same raw video and denoised audio path, but with reviewed decisions.
  • A separate diff endpoint strikes out both editors' cuts on the raw transcript. Pipeline cuts come from the EDL. Human cuts come from word-level alignment with a new transcript of the human-edited video.

Production boundary

  • This transcript drives editing and review. It is not the final transcript shown to Gradivo students.
  • Gradivo should transcribe the final reviewed Mux video again. This keeps chapters, explanations, and timestamps aligned with the published video.
  • The AI editor currently uses local file paths and companion files. It does not yet use permanent Gradivo job IDs.
  • Gradivo still owns the raw-recording link, problem mapping, publishing status, final Mux video, chapters, explanations, and Notion sync.

Known technical gaps before production service use

  • The section editor is on by default, but the latest 98-video review artifact was recalculated from older Sol output. It is not a fresh current-main run.
  • Clean Fizika and Hrvatski videos still score poorly. The saved Fizika cohort has 0.304 precision, 0.176 recall, 0.223 F1, and 80 overcut words.
  • We need stronger proof that partial false-start, stutter, and filler cuts are safe on lessons that need few edits.
  • Render defaults are for development and still need production validation.
  • Intro and outro insertion is not implemented.
  • Export to a professional non-linear editor (NLE) is planned but not implemented.
  • Professor-specific settings and trained custom models are future work.
  • A production service still needs permanent job IDs, authenticated file access, queues, retries, cancellation, progress and failure states, and storage adapters.
