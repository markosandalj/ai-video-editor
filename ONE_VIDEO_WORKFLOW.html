The actual path

What happens to one recording

There is one active AI text-editing path. The old duplicate cascade, enrichment judge, sentence badges, heat map, and restore queue are not part of this flow.

Step by step

The six stages

These stages describe the active path for a single raw recording. Batch processing repeats the same path for each video.

Prepare the audio

The system extracts audio, reduces steady background noise, finds silences of at least three seconds, and measures loud non-speech disruptions such as coughs or microphone bumps.

  • The original video picture is left untouched.
  • Long silence becomes an initial keep/cut outline.
  • Audio evidence can help identify a stranded false start.

Create a timed transcript

ElevenLabs Scribe turns Croatian speech into words with timestamps. Long pauses can split a sentence into smaller editing units. Gemini then corrects only obvious spelling mistakes; it does not rewrite the professor's style.

  • Every word remains tied to its original place in the video.
  • Spoken fillers and imperfect grammar are preserved.
  • The transcript is cached so experiments do not need new transcription.

Decide what to cut

GPT-5.6 Sol reads roughly 1,200 words at a time and returns exact transcript spans for retakes, false starts, stutters, or filler. Code verifies each span and converts it to timestamps. A narrow deterministic detector also catches exceptionally clear local correction chains.

  • Sol is the main and only active text-judgment editor.
  • Risky "redundant" removals are rejected and remain kept.
  • Audio false starts and short off-lesson asides run as supporting lanes.

Build and render the automatic draft

Accepted decisions are merged into one edit decision list, or EDL. Each cut keeps its source reason, confidence, and note. Cut edges are moved to nearby quiet audio points before the automatic MP4 is rendered.

  • Whole sentences and partial word spans can both be cut.
  • Real silence remains distinct from AI text cuts.
  • Short audio fades soften the joins.

Review and correct the timeline

The review app shows the video, waveform, transcript, and cut ranges. The reviewer can select any source-time range to cut, restore existing cuts, preview edited playback, save, and move between videos.

  • Transcript and timeline are two views of the same ranges.
  • The Compare screen shows pipeline versus human cuts for QA fixtures.
  • There are no badges, heat maps, salience scores, or restore queue.
  • The reviewed video can be rendered from the same app.

Save and render the reviewed result

Review changes are saved as a separate reviewed EDL. Rendering uses the raw video, cleaned audio, and reviewed keep ranges to create a separate reviewed MP4.

  • The automatic EDL remains available for comparison.
  • An empty review cut list restores the entire source.
  • The reviewed render never overwrites the raw recording.

Evaluation adds more evidence

QA runs use cached raw transcripts and human-edited transcripts. They save per-video EDLs, every section-editor proposal and its accepted or rejected outcome, model and prompt metadata, aggregate scores, and reference comparisons.

  • Runs checkpoint after each video and can resume.
  • Precision measures how often proposed cuts were right.
  • Recall measures how many human cuts the pipeline found.
  • The Compare screen exposes individual disagreements.

Human review

What the reviewer is checking

The reviewer does not need annotation scores. The transcript and timeline show what will actually be removed, and the reviewer checks that the lesson still makes sense and sounds natural.

Keep the lesson intact

  • Was a useful explanation cut? Restore it.
  • Was an incorrect take kept? Cut it.
  • Did a correction get handled properly? Keep the correct version.
  • Does the math or explanation still flow? Check before approving.

Keep the video natural

  • Is a pause too short? Adjust the decision if it feels rushed.
  • Does a transition sound awkward? Review the nearby cut.
  • Is a short filler phrase actually natural? It can stay.
  • Does the waveform hide an awkward edge? Move the cut range.

Predictable boundaries

What the AI decides and what code decides

Sol makes the semantic judgment about which spoken text looks removable. Code then constrains that judgment before it can change the timeline.

Automatic protections

  • The text must exist: invented or badly matched spans are rejected.
  • The section must own it: overlap context cannot double-cut it.
  • Short whole-sentence retakes are protected: recurring lines such as "Dobro" are normally kept.
  • Far-away retakes are recap risks: supplied evidence more than 60 seconds away is rejected.
  • "Redundant" content stays: this risky deletion type is rejected, not queued for annotation review.

What the protections do not guarantee

  • A verbatim span can still be the wrong editorial choice.
  • Partial false starts, stutters, and filler currently have fewer semantic safeguards than whole retakes.
  • On an already-clean lesson, a few plausible AI guesses can dominate the small number of real cuts.
  • Human review is still required before the automatic result is final.
  • QA compares every change against the same 98 human-edited fixtures.

Current quality status

What we know today

The latest complete corpus artifact is useful evidence, but it is not a fresh run of every change now on the main branch.

What passed

  • The section editor replaced the old tiered duplicate path as the default.
  • The latest evaluated 98-video artifact scored about 0.797 cut precision, 0.676 recall, and 0.732 F1.
  • The deterministic correction lane added 33 correct cut words and zero overcuts in its fixed-EDL projection.
  • Enrichment and its arbiter were removed because they did not reliably improve the new cutter.

What still needs work

  • The 98-video review workspace uses older Sol EDLs plus the Iteration 19 projection, not a fresh current-main run.
  • Fizika is a weak spot: the reference cuts only about 1.8% of its words, so unnecessary partial cuts hurt badly.
  • On the saved Fizika cohort, only 35 of 115 pipeline-cut words match the reference; 80 are scored as overcuts.
  • The next honest measurement should rerun the clean-video cohort with current provenance and traces before another full corpus run.

Production boundary

Where this process ends

The AI video editor prepares and reviews the edited video. Gradivo still owns the final publishing workflow.

AI video editor owns

  • Temporary media needed for processing.
  • Speech analysis and editing transcript.
  • Automatic EDL and acoustic boundary snapping.
  • Browser-based transcript and timeline review.
  • Separate automatic and reviewed renders.
  • Fixture QA, traces, and model comparisons.

Gradivo still owns

  • Which raw recording belongs to which problem.
  • Production status and responsibility tracking.
  • Upload to the final video platform.
  • Student-facing transcript, chapters, and explanations.
  • Final review, publishability, and publishing.

The editing transcript is not the final student transcript

The transcript created during AI editing helps the system make and review cut decisions. Gradivo should still create the student-facing transcript from the final reviewed video, so the transcript matches exactly what students watch.

Core principle: AI proposes exact cuts, code constrains them, and a human owns the final timeline.
